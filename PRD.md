# AI Daily Pulse - 银行后台业务AI赋能助手

## 产品需求文档 (PRD)

---

## 一、项目背景与问题陈述

### 1.1 目标用户画像


| 属性   | 描述                      |
| ---- | ----------------------- |
| 岗位   | 银行资金同业交易后台              |
| 系统   | 静态数据维护系统                |
| 业务领域 | 资金业务、KYC/AML、风险管理、运营自动化 |
| 痛点   | 日常事务繁重，缺乏获取AI赋能信息的渠道    |


### 1.2 核心痛点

1. **静态数据维护繁重** - 大量重复性的数据录入、核对、更新工作
2. **风险合规压力大** - 资金交易涉及大量合规检查，人工核对效率低
3. **跨系统数据孤岛** - 不同系统间数据不一致，需要大量手工协调
4. **AI应用信息闭塞** - 难以系统性了解AI技术在金融行业的最新应用案例
5. **同行动态不了解** - 不清楚其他商业银行AI赋能的进度和实践
6. **业务灵感匮乏** - 每天忙于日常事务，没有时间和渠道获取业务创新灵感
7. **技术落地困难** - 看到好的AI应用案例，不知道如何与自身业务结合
8. **知识难以复用** - 缺乏系统性的历史记录，无法复盘和持续优化

## 二、产品解决方案

### 2.1 核心功能

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AI Daily Pulse - 每日AI赋能报告                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  📰 板块一：AI最新动态                                              │
│     - 今日AI圈发生的重大事件                                         │
│     - 大模型更新、产品发布                                           │
│     - 开源项目动态                                                   │
│                                                                     │
│  💰 板块二：金融AI应用（重点）                                      │
│     - 国内外银行AI应用案例                                          │
│     - 金融科技最新进展                                              │
│     - 行业报告和趋势分析                                            │
│                                                                     │
│  💡 板块三：业务赋能建议                                            │
│     - 基于今日资讯生成的3-5条业务建议                                │
│     - 每个建议包含：标题、描述、落地可行性、实施难度                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 解决方案描述

1. **多源资讯统一抓取** - 自动从Twitter、Reddit、Hacker News、TechCrunch、虎嗅、机器之心、量子位等平台抓取AI资讯
2. **金融AI应用聚焦** - 专门搜集金融科技资讯、银行AI应用案例、行业报告
3. **智能业务建议生成** - 基于每日资讯，使用LLM生成可落地的业务赋能建议
4. **微信推送** - 每天早上8:30自动推送到微信
5. **历史存档与复盘** - 所有内容本地SQLite存储，支持回顾和反馈优化

### 3.3 业务建议生成需求

#### 提示词模板

```
# 角色
你是一位深耕金融行业的AI业务顾问，专门为银行后台业务人员提供AI赋能建议。

## 背景
- 用户岗位：资金同业交易后台
- 系统类型：静态数据维护系统
- 业务领域：资金业务、KYC/AML、风险管理、运营自动化

## 今日资讯摘要
{今日抓取的AI资讯和金融AI应用案例摘要}

## 任务
根据今日AI资讯和金融AI应用案例，生成3-5条可落地的业务赋能建议。

## 输出格式
每个建议包含：
- 建议标题（简洁有力，不超过20字）
- 建议描述（为什么有用、如何落地，50字以内）
- 实施难度（低/中/高）
- 预期收益（量化描述）

## 约束
1. 建议必须与银行后台业务直接相关
2. 考虑静态数据维护系统的特点
3. 优先考虑低实施难度、高收益的建议
4. 避免过于技术化的建议
5. 建议要具体可执行，不能是空泛的概念
```

#### 输出示例

```
💡 建议1：智能合同审阅助手
   描述：利用OCR+NLP自动识别合同关键条款，减少人工核对时间
   实施难度：低（可采购成熟产品）
   预期收益：合同审核效率提升50%

💡 建议2：交易对手画像自动化
   描述：基于公开数据自动生成交易对手风险画像，辅助KYC
   实施难度：中
   预期收益：尽调时间缩短30%
```

## 五、数据模型

### 5.1 数据库表结构

```python
# news_items - 资讯条目
{
    id: int (PK)
    source: str          # 来源：twitter, reddit, huxiu, weidaoke...
    source_name: str     # 来源名称
    title: str           # 标题
    url: str (unique)   # 原文链接
    summary: str         # 摘要
    content: str         # 完整内容
    author: str          # 作者
    published_at: datetime # 发布时间
    category: str       # 分类：产品发布/技术研究/金融应用...
    is_financial: bool   # 是否金融相关
    created_at: datetime
}

# business_ideas - 业务建议
{
    id: int (PK)
    date: date           # 日期
    idea_number: int    # 建议序号（1-5）
    title: str          # 建议标题
    description: str    # 建议描述
    difficulty: str     # 实施难度：低/中/高
    expected_benefit: str # 预期收益
    source_items: str   # 关联的资讯ID（JSON数组）
    created_at: datetime
}

# feedback - 用户反馈
{
    id: int (PK)
    idea_id: int (FK)   # 关联的建议ID
    feedback_type: str  # 有用/没用/太复杂/需要更多细节
    user_comment: str   # 用户补充评论
    created_at: datetime
}

# daily_reports - 每日报告
{
    id: int (PK)
    date: date (unique) # 日期
    content: str       # 完整的Markdown内容
    created_at: datetime
}
```

## 六、非功能性需求

### 6.1 性能需求

- 单次抓取完成时间 < 5分钟
- LLM生成建议时间 < 30秒
- 内存占用 < 500MB

### 6.2 稳定性需求

- 单一数据源失败不影响整体流程
- 网络异常时记录日志并重试
- LLM API失败时使用缓存或跳过

### 6.3 安全需求

- API Keys存储在.env文件，不提交到代码仓库
- 微信Webhook URL仅本地保存
- 定期清理过期的临时数据
